# MCP Notes Server

Навчальний MCP-сервер (тема **Notes**) для роботи з нотатками просто з Claude
Desktop. Сервер написаний на Python з офіційним пакетом [`mcp`](https://pypi.org/project/mcp/),
працює через transport **stdio** і зберігає нотатки в локальному JSON-файлі —
це реальна персистентна логіка, без зовнішніх API чи ключів.

## Вимоги

- Python **3.10+**
- [`uv`](https://docs.astral.sh/uv/) (рекомендовано для запуску й керування залежностями)
- Claude Desktop (для підключення сервера)

## Встановлення

```bash
# 1. Клонувати репозиторій і перейти в директорію
git clone <repo-url> lesson-16.MCP
cd lesson-16.MCP

# 2. Встановити залежності (uv створить .venv і поставить mcp)
uv sync

# 3. (опційно) Перевірити, що сервер запускається
uv run server.py        # запускає stdio-сервер; Ctrl+C для виходу
```

> Сервер за замовчуванням зберігає нотатки у файлі `notes_data.json` поряд із
> `server.py`. Шлях можна змінити змінною середовища `NOTES_DATA_FILE`.

## Підключення до Claude Desktop

1. Відкрити файл конфігурації Claude Desktop:
   - **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
   - **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`
2. Додати блок `mcpServers` (див. приклад у [`claude_desktop_config.json`](claude_desktop_config.json)):

   ```json
   {
     "mcpServers": {
       "notes": {
         "command": "/opt/homebrew/bin/uv",
         "args": ["--directory", "/Users/kostromin/lesson-16.MCP", "run", "server.py"]
       }
     }
   }
   ```

   > ⚠️ Вкажіть **абсолютні** шляхи: Claude Desktop запускається з мінімальним
   > `PATH` і може не знайти `uv` за коротким іменем. Дізнатися шлях до `uv`:
   > `which uv`. Замініть `--directory` на абсолютний шлях до цієї директорії.
3. **Повністю перезапустити** Claude Desktop (Quit, не просто закрити вікно).
4. У новому чаті має з'явитися сервер `notes` з доступними інструментами 🔨.

## Tools

| Tool | Аргументи | Обов'язковість | Опис |
|------|-----------|----------------|------|
| `create_note` | `title: str`, `content: str`, `tags: list[str] = None` | `title`, `content` — **обов'язкові**; `tags` — опційний | Створити нову нотатку |
| `search_notes` | `query: str = ""`, `tag: str = ""` | **обидва опційні** (порожні → усі нотатки) | Пошук за текстом і/або тегом |
| `update_note` | `note_id: str`, `title?`, `content?`, `tags?` | `note_id` — обов'язковий; решта — опційні | Оновити обрані поля нотатки |
| `delete_note` | `note_id: str` | `note_id` — **обов'язковий** | Видалити нотатку |

## Resources

| Resource | Тип | Опис |
|----------|-----|------|
| `notes://all` | static | Усі збережені нотатки |
| `notes://stats` | static | Кількість нотаток і розподіл за тегами |
| `notes://{note_id}` | template | Конкретна нотатка за ідентифікатором |

## Приклади діалогів

### Приклад 1 — створення нотатки (`create_note`, required args)

> **Користувач:** Створи нотатку «Список покупок» з текстом «молоко, хліб, кава» і тегом shopping.
>
> **Claude** *(викликає `create_note(title="Список покупок", content="молоко, хліб, кава", tags=["shopping"])`)*
>
> ✅ Нотатку створено: `note-1` — «Список покупок».

### Приклад 2 — пошук і читання ресурсу (`search_notes` + `notes://all`)

> **Користувач:** Покажи всі нотатки з тегом shopping.
>
> **Claude** *(викликає `search_notes(tag="shopping")`)*
>
> Знайдено нотаток: 1
> ```
> [note-1] Список покупок
> Теги: shopping
> Створено: 2026-05-31T12:00:00+00:00 | Оновлено: 2026-05-31T12:00:00+00:00
> ---
> молоко, хліб, кава
> ```

### Приклад 3 — видалення нотатки (`delete_note`, required arg)

> **Користувач:** Видали нотатку note-1.
>
> **Claude** *(викликає `delete_note(note_id="note-1")`)*
>
> 🗑️ Нотатку видалено: `note-1` — «Список покупок».

> Скріншоти цих діалогів у Claude Desktop — у директорії [`examples/`](examples/).

## Known limitations

- **Локальне сховище:** нотатки зберігаються у JSON-файлі на тій машині, де
  запущено сервер. Немає синхронізації між пристроями.
- **Без блокувань файлу:** запис не захищений від одночасного доступу кількох
  процесів — припускається один екземпляр сервера.
- **Простий пошук:** `search_notes` робить підрядковий пошук без урахування
  морфології, релевантності чи full-text індексу.
- **Лічильник id залежить від файлу:** ідентифікатори (`note-N`) генеруються з
  `next_id` у сховищі; видалення файлу скидає нумерацію.
- **Немає пагінації:** `notes://all` і `search_notes` повертають усі результати
  одразу — на дуже великих обсягах відповідь може бути довгою.
- **Без автентифікації/авторизації:** будь-хто з доступом до сервера має повний
  доступ до нотаток.
